Mit dem SCUI Simple Consol Unser Interface steht eine einfache Benutzerschnittstelle für ein Standalone Java Programm zur Verfügung.

Überischt

Mit dem SCUI Simple Consol Unser Interface steht eine einfache Benutzerschnittstelle für ein Standalone Java Programm zur Verfügung.

Funktionen

  • Es lassen sich einfache Menues darstellen
  • Auf jeder Menuposition kann eine Callback Funktion in das zu steuernde Programm aufgeruffen werden.
  • Es lassen sich Content Inhalte auf der Konsole darstellen
  • Mit AutoRun kann ein automatisiertes Verarbeiten konfiguriert und gestartet werden.

Anwendung

1) Registrieren und starten

Die SCUI lässt sich sehr einfach registrieren und starten. Entweder mit oder ohne AutoRun Funktion. Die Definition und Gestaltung vom UserInterface wird in der CUIImplementierung hinterlegt. Wenn beim Erzeugen vom SCUI eine String[] übergeben wurde, werden die dort aufgeführten Benutzereingaben automatisch ausgeführt.  
   SCUI scui = new SCUI();
   scui.work(new CUI_Implementierung(scui, this));
   SCUI scui = new SCUI(new String[]{"Display","Run"});
   scui.work(new CUI_Implementierung(scui, this));

2) UI Definieren und Menu-Funktionen (Callback) implementieren

CUI ist ein Interface. Bei der Implementierung vom CUI (Consolen User Interface) ist die Implementierung von zwei Methoden zwingend.
Es sind diese.
  • CUIMenu  getUIMenu();  
  • CUITabelle  getContentTabelle();
public class CUIMenue implements CUI {

      private Run run;
      private SCUI scui;

      public CUIMenue(SCUI scui, Run run) {
         this.run = run;
         this.scui = scui;
      }
...

Ein CUIMenu enthält IOElemente. Ein IOElemente  besteht aus einem Funktions oder Menu Code, einem beschreibenden Text und einer Callback-Methode die den Codierten Menupunkt abarbeitet.
Beispiel:
   public CUIMenu getUIMenu() {
      CUIMenu menu = new CUIMenu();      
      menu.addIOElement(new IOElement("Code", "Funktion", new NotTodo()));
      menu.addIOElement(new IOElement("----", "--------", new NotTodo()));
      menu.addIOElement(new IOElement("E1", "Edit CompusetFilesPath", new E1()));
      menu.addIOElement(new IOElement("E2", "Edit EasyDocFilePath", new E2()));
      menu.addIOElement(new IOElement("E3", "Edit EasyDocFileName", new E3()));
      menu.addIOElement(new IOElement("E4", "Edit OutputFilePath", new E4()));
      menu.addIOElement(new IOElement("E5", "Edit SprachCode", new E5()));
      menu.addIOElement(new IOElement("R", "Start Run", new Generate()));
      menu.addIOElement(new IOElement("<", "Ende", new E()));
      return menu;
   }


Eine CUITabelle ist eine mehrspaltige String Tabelle die rechts vom Menue als Content dargestellt wird.
Beispiel:

   public CUITabelle getContentTabelle() {
      CUITabelle t = new CUITabelle();
      t.addZeile(new String[] {"Status", "Data", " ", "Inhalt"});
      t.addZeile(new String[] {"------", "----", " ", "------"});
      t.addZeile(new String[] {(run.checkCompusetFilesPath()) ? "ok" : "Fehler", "CompusetFilesPath", "=", run.getCompusetFilesPath()	});
      t.addZeile(new String[] {(run.checkEasyDocFilePath()) ? "ok" : "Fehler", "EasyDocFilePath", "=", run.getEasyDocFilePath()});
      t.addZeile(new String[] {(run.checkEasyDocFileName()) ? "ok" : "Fehler", "EasyDocFileName", "=", run.getEasyDocFileName()});
      t.addZeile(new String[] {(run.checkOutputFilePath()) ? "ok" : "Fehler", "OutputFilePath", "=", run.getOutputFilePath()});
      t.addZeile(new String[] {(run.checkSprachCode()) ? "ok" : "Fehler", "SprachCode", "=", run.getSprachCode()});
      return t;
   }


Menu Funktionen -> Callback Methoden

Nachfolgend ist ein Beispiel eine Callback Funktion implementiert. 

   class E1 extends Call {
        public boolean run() {
            run.setCompusetFilesPath(scui.inputText("CompusetFilesPath Eingeben"));
            scui.setContent(getContentTabelle());
            return false;
        }
    }
